# Symptoms-to-Sales 2.0: Implementation Plan

> Reference document for building consolidated SaaS platform
> Based on Tony Teegarden call (Dec 4, 2024)

---

## Executive Summary

Upgrade the current 7-tool Pickaxe-based S2S suite into a consolidated, vibe-coded Next.js application with Claude API backend. Focus on user experience, context switching between personal/partner work, and leveraging Claude's superior writing quality.

**Key Pain Point (from Tony):** Current Pickaxe implementation feels like "forcing a cow through a straw" - outputs don't match what Claude can actually produce.

---

## Tech Stack

| Layer | Technology | Rationale |
|-------|------------|-----------|
| Frontend | Next.js 14+ (App Router) | Full control, server components for API security |
| Database | Supabase (PostgreSQL) | Auth, storage, row-level security |
| AI | Claude API (Anthropic) - Sonnet 4.5 | Superior writing quality vs. Pickaxe |
| Styling | Tailwind CSS 4 + shadcn/ui | Rapid professional UI |
| Hosting | Vercel | Seamless Next.js deployment |

## UI/UX Design System

### Design Philosophy: "The Writer's Studio"

This is NOT generic SaaS. This is a precision instrument for deal-makers who care about craft.

**Core Principles:**
1. **Content is the Hero** - Generated copy commands visual presence. Inputs are secondary.
2. **Effortless Context** - Project switching feels invisible until needed.
3. **Controlled Magic** - AI generation feels like watching a master at work, not a loading spinner.
4. **Writerly, Not Corporate** - Editorial feel, not enterprise software.
5. **Deliberate Restraint** - Every element earns its place.

**Inspiration:** Linear's precision, iA Writer's typography, Stripe's micro-detail, the feeling of a well-appointed writing desk.

```typescript
const systemPrompt = `
${TRAVIS_IP_FOUNDATION}  // Core methodology - always present

${TOOL_SPECIFIC_PROMPT}  // From existing prompt files

${project.voiceDNA?.content || 'Use conversational Travis-style voice'}

## OFFER CONTEXT
${JSON.stringify(project.offerContext)}

## FORBIDDEN PATTERNS
- No "No X. No Y. Just Z." setup
- No staccato short fragments
- No triplets ("Simple. Easy. Fast.")
- No corporate transitions (Moreover, Furthermore)
```
